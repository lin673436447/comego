# "往来记"产品功能设计

## 一、核心功能模块

### 1. 人情记账
- **快速记账**：单条记录快速录入，支持金额、对象、类型、日期、备注等基本信息
- **批量记账**：多条记录同时录入，适用于婚礼、生日等场景
- **收支区分**：支持收入和支出两种记账方向
- **支付方式**：支持现金和礼物两种支付方式
  - 现金方式：填写金额
  - 礼物方式：填写礼物名称和礼物价值（可选）
- **分类管理**：预设常见人情类型（婚礼、生日、节日、乔迁、聚餐、活动等）
- **往来对象**：自动保存常用联系人，支持快速选择
- **图片附件**：可添加票据或场景照片作为凭证
- **语音输入**：支持语音快速记录，提高操作效率

### 2. 礼簿管理
- **创建礼簿**：支持创建婚礼、生日、葬礼、其他类型的礼簿
- **礼簿信息**：礼簿名称、类型、日期、描述等基本信息
- **礼簿记账**：在礼簿内添加多条人情记录
- **礼簿统计**：自动计算总收入、总支出、总金额
- **礼簿查询**：按时间、类型、金额等多维度筛选礼簿
- **礼簿导出**：支持导出Excel或PDF格式的礼簿数据
- **礼簿分享**：支持分享礼簿给其他用户（高级功能）

### 3. 账单明细
- **明细查询**：按时间、对象、类型、收支方向等多维度筛选
- **往来历史**：查看与特定对象的历史往来记录
- **余额提醒**：智能计算与他人的人情往来余额，提醒是否需要回礼
- **数据导出**：支持导出Excel或PDF格式的账单明细

### 4. 数据分析
- **收支统计**：按月份、季度、年度统计人情收支
- **分类分析**：展示不同类型人情的支出占比
- **支付方式分析**：现金和礼物的使用情况统计
- **关系图谱**：可视化展示人情往来的网络结构
- **预算管理**：设置人情支出预算，超支提醒
- **礼簿分析**：各礼簿的收支情况对比分析

### 5. 个人中心
- **个人信息**：基本资料管理
- **隐私设置**：数据加密、隐私保护选项
- **消息通知**：重要日期提醒（如朋友生日、纪念日）
- **会员等级**：普通会员和高级会员功能区分


## 二、产品迭代路线

### 第一阶段（MVP）
- 核心记账功能（快速记账、批量记账）
- 礼簿创建和管理
- 基本查询统计
- 收入/支出区分
- 现金/礼物支持

### 第二阶段
- 高级数据分析
- 人情计算器
- 回礼提醒
- 多端同步
- 礼簿分享功能

### 第三阶段
- 社交功能
- 个性化推荐
- 更多场景化功能
- 商业合作拓展

## 五、技术架构

### 后端技术栈
- **核心框架**：Spring Boot
- **ORM框架**：Mybatis-Plus
- **数据库**：MySQL
- **构建工具**：Maven
- **JDK版本**：JDK8

### 前端技术栈
- **基础技术**：uniapp（支持微信小程序、H5、App）
- **UI组件**：Element UI等第三方组件
- **状态管理**：Vuex
- **网络请求**：uni.request

### 数据库设计
- **用户信息表**：存储用户基本信息
- **联系人表**：存储联系人信息
- **礼簿表**：存储礼簿信息
- **人情记录表**：存储人情往来记录
- **预算设置表**：存储预算配置
- **余额记录表**：存储与他人的往来余额
- **通知提醒表**：存储提醒信息

## 六、总结

"往来记"产品通过聚焦人情往来这一细分场景，解决了目标用户群体的实际痛点。产品设计坚持"简单易用、礼簿一目了然"的原则，通过直观的界面和智能的功能，帮助用户轻松管理人情往来，建立健康的社交关系。

产品核心创新点：
1. **礼簿概念**：将分散的人情记录整合到礼簿中，便于管理和统计
2. **收支区分**：清晰区分收入和支出，便于人情往来管理
3. **多种支付方式**：支持现金和礼物，更贴近实际使用场景
4. **批量记账**：提高记账效率，适用于婚礼、生日等场景

通过持续的功能迭代和用户体验优化，有望成为人情管理领域的标杆产品。
